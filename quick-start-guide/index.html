Checking User Credentials​

For more detailed information, please see the Check User Credentails section of the API documentation.

The second step of enabling 1-Click or Free is to check if the user has the necessary credentials to complete your signup flow. To do this, call the /hasMatchingCredentials endpoint with the user's email and/or phone along with the credential type(s) you are looking for.

Two outcomes are possible with their respective response bodies below:

  1. The user has all the required credentials. The response body contains a url attribute that you should direct the user to.
Example Match Response Body
{
    "match": true,
    "url": "https://wallet-beta.unumid.co/request/9f2053fc-cc4c-41cb-8d14-7d3ccf6167eb"
}
  1. The user does not have all the required credentials. The response body contains a match value of false.
Example Not a Match Response Body
{
    "match": false,
    "url": ""
}

(1) The User Has the Credentials​

If the user has all the required credentials, you need to direct the user to the url provided in the response body. (This is a link to the Unum ID Wallet with parameters.) The user will be prompted to authenticate and share the credentials you've requested. When they agree to share, a sharedCredentialsUuid is returned to your client via a query param. Please proceed to the Get Shared Credentials section.

(2) The User Does Not Have the Credentials​

If the user does not have the credentials, you should take the user through your existing signup flow to collect and verify the necessary data. Once the verified information is obtained, you need to issue the user credentials containing that data. This can be done completely behind the scenes, as long as you obtain user consent. Please proceed to the Issuing Credentials section.

An example response body looks like:

Example Get Shared Credentials Response Body
{
    "uuid": string // the sharedCredentialsUuid
    "credentials": [{
        "id": "8a1d4e35-413d-496b-b499-8810b55cfb5c",
        "type": "SsnCredential",
        "data": {
            "ssn": "111-22-3333"
        },
        "issuanceDate": "1671847264479",
        "expirationDate": "1871839024044",
        "issuer": "32ef9562-312d-4a8b-8f3e-ccb8df500b55",
    }],
    "email"?: string // the user's email from the input to /hasMatchingCredentials; only present if email was provided
    "phone"?: string // the user's phone from the input to /hasMatchingCredentials; only present if phone was provided
}

To facilitate extracting the credential data, we recommend leveraging the Schema Resolver to get the schema of the credential types. Because credential schemas are immutable, they can then be cached and referenced when retrieving the credential data.

When you extract the data, you now have the necessary information to consider the user fully verified. (You can of course perform additional verification checks as you like.)

Issuing User Credentials​

For more detailed information, please see the Issue Credentails section of the API documentation.

Once you have collected and verified the user's information, you can issue credentials. This can be done by calling the /credentials endpoint with the user's verified email address and/or phone along with the relevant credential type and credential data. The credential data must match the schema of the credential type you are issuing.

In doing so, you qualify for reimbursement from Unum ID, and you stand to gain future revenue when the user shares those credentials with other brands.

For example, if you collected the user's SSN, you can issue the user a SsnCredential by calling the /credentials endpoint with the user's email and/or phone along with the SsnCredential type and the SSN data:

Example Issue Credentials Request Body
{
  "email": "test@unumid.abcde",
  "phone": "+421234567890",
  "credentials": [
    {
      "type": "SsnCredential",
      // data format following the properties attribute from https://schema.unumid.co/jsonSchema/SsnCredential
      "data": {
        "ssn": "111-22-3333"
      },
      "expirationDate": 1893484800
    }
  ]
}

The format of data was from the json.properties attribute of the response from the Schema Resolver for the SsnCredential, i.e. https://schema.unumid.co/schema/SsnCredential.

Upon issuing credentials you qualify for identity verification monetary reimbursement from Unum ID. Also, you stand to gain future revenue when the user shares those credential with other brands. Congratulations! You have now enabled 1-Click or Free ID Verification.
