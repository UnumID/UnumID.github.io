Usage Guide

Write content using GitHub-flavored Markdown syntax.

Code#

Syntax Highlighting#

For example, this...

```javascript
var s = 'JavaScript syntax highlighting';
alert(s);
```

...renders to:

var s = 'JavaScript syntax highlighting';
alert(s);

Line Highlighting#

Use e.g. {2} after the langauge name (and a space):

function highlightMe() {
console.log('This line can be highlighted!');
}

Tabs#

Tabs can be used generally, but they're useful for language selection:

  • JavaScript
  • Python
function helloWorld() {
console.log('Hello, world!');
}
def hello_world():
print 'Hello, world!'

To make multiple tab blocks update in sync, give all related tabs the same groupId prop. Note that doing this will persist the choice in localStorage and all <Tab> instances with the same groupId will update automatically when the value of one of them is changed. Note that groupId are globally-namespaced.

  • JavaScript
  • Python
function goodbyeWorld() {
console.log('Goodbye, world :(');
}
def goodbye_world():
print 'Goodbye, world :('

Tooltips#

We can use tooltips like this to provide quick, inline definitions and explanations.

Using these is super simple. Add these lines at the top of a .mdx file (below the file metadata):

example.mdx
import TooltipWithDefaults from '../src/components/TooltipWithDefaults';
import Tip from '../src/components/Tip';

And then you're ready to go. There are two kinds of tooltip:

  1. local (for one-time use), and
  2. global (for multiple uses).

Local#

Local tooltips are defined on the spot, so they're best for one-time use. Just use a <Tip> element and set the text attribute. For example, this...

<Tip text="Super helpful tooltip!">display text</Tip>

... renders to: display text.

Global#

Global tooltips are defined globally so they can be used multiple times. This is especially useful for definitions. Just use an empty <Tip> element and set the type attribute (the display text is part of the definition). For example, this...

<Tip type="credential"/>

... renders to:

A credential is a collection of data about a person. It's issued by a company (i.e. created and sent to a user) and stored in the company's app, on that user's device.
+ More...
Example: ACME Bank issues a KYC verification credential to Richard (an ACME user). This includes Richard's contact information and account numbers, as well as a level of confidence in the accuracy of the data.
Components: A company issues credentials using the Server SDK, and an app stores credentials using the Mobile SDK.
credential. (Setting the text attribute or putting text in between <Tip></Tip> tags does nothing.)

To define a global tooltip, add an entry to the tooltips object in /src/components/Tip.jsx like so:

const tooltips = {
"display text": "tooltip text",
}

You can use a simple string like above, but you can also use any HTML to make much richer tooltips with multiple lines, formatting, links, etc.
