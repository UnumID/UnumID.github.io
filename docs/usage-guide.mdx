## Code

### Syntax Highlighting

For example, this...

    ```javascript
    var s = 'JavaScript syntax highlighting';
    alert(s);
    ```

...renders to:

```javascript
var s = 'JavaScript syntax highlighting';
alert(s);
```

### Line Highlighting

Use e.g. `{2}` after the langauge name (and a space):

```js {2}
function highlightMe() {
  console.log('This line can be highlighted!');
}
```

### Tabs

Tabs can be used generally, but they're useful for language selection:

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs
  groupId="languages"
  defaultValue="js"
  values={[
    { label: 'JavaScript', value: 'js', },
    { label: 'Python', value: 'py', },
  ]
}>
<TabItem value="js">

```js
function helloWorld() {
  console.log('Hello, world!');
}
```

</TabItem>
<TabItem value="py">

```py
def hello_world():
  print 'Hello, world!'
```

</TabItem>

</Tabs>

To make multiple tab blocks update in sync, give all related tabs 
the same `groupId` prop. Note that doing this will persist the 
choice in `localStorage` and all `<Tab>` instances with the 
same `groupId` will update automatically when the value of 
one of them is changed. Note that `groupId` are globally-namespaced.

<Tabs
  groupId="languages"
  defaultValue="js"
  values={[
    { label: 'JavaScript', value: 'js', },
    { label: 'Python', value: 'py', },
  ]
}>
<TabItem value="js">

```js
function goodbyeWorld() {
  console.log('Goodbye, world :(');
}
```

</TabItem>
<TabItem value="py">

```py
def goodbye_world():
  print 'Goodbye, world :('
```

</TabItem>

</Tabs>

### Live Editor

You can make a live editor! Just add `live` after the language name:

```jsx live
function Clock(props) {
  const [date, setDate] = useState(new Date());
  useEffect(() => {
    var timerID = setInterval(() => tick(), 1000);

    return function cleanup() {
      clearInterval(timerID);
    };
  });

  function tick() {
    setDate(new Date());
  }

  return (
    <div>
      <h2>It is {date.toLocaleTimeString()}.</h2>
    </div>
  );
}
```

| Tables        |      Are      |   Cool |

## Tooltips

We can use tooltips like <u data-tip="Wow, a tooltip!">this</u>
with the help of [`react-tooltip`](https://www.npmjs.com/package/react-tooltip).

I'm not sure how to enable tooltips globally, so for now at least 
we have to add them to each `.mdx` page and set the same defaults
there like so:

```jsx title="example.mdx"
import ReactTooltip from 'react-tooltip';

<ReactTooltip 
  type="dark"
  place="top"
  effect="solid"
  multiline={true} // use <br> e.g. data-tip="Multiple <br> lines!"
  textColor="white"
  backgroundColor="black"
  border={true} // helpful for dark mode
  borderColor="white"
  delayHide={250}
  resizeHide={false} // don't hide when resizing page
/>
```

Many of these are defaults of the `react-tooltip` package, but it's useful to make them explicit here as a reminder of the most relevant options.

To make a tooltip, use an HTML `<u>` element (you can add it inline within the `.mdx` file) and set the `data-tip` attribute. For example:

```html live
<u data-tip="This is a tooltip!">hover over me</u>
```

Avoid using other elements like `<div>`, which fills the width of the parent element because they have `display: block;`, which makes the tooltip appear in the middle of the parent element. You can also use an element like `<span>`,but using `<u>` creates an underline, which is a nice visual indicator that there's a tooltip, so we should use that by convention.

You can make multiline tooltips using `<br>`. For example:

```html live
<u data-tip="Multiple <br> lines <br> ðŸ¤©">hover over me</u>
```

And you can set override the default options for individual tooltips. For example:

```html live
<u 
  data-tip="Super duper custom tooltip... <br> I take forever to disappear"
  data-background-color="green"
  data-delay-hide="5000"
>hover over me</u>
```

Very helpfully, you can also set a single tooltip to be used multiple times. To do so, you use a `ReactTooltip` component and set an `id` for it. Then, you leave `data-tip` unspecified (but you still need to include it) and set `data-for` to the `id` of the component. For example, if you define this...

```jsx
<ReactTooltip id="gobbledygook"
  type="dark"
  place="top"
  effect="solid"
  multiline={true} // use <br> e.g. data-tip="Multiple <br> lines!"
  textColor="white"
  backgroundColor="#5e5e5e"
  delayHide={250}
  resizeHide={false} // don't hide when resizing page
>
  Random <br></br> <strong>malarkey!</strong>
</ReactTooltip>
```

... you can then create multiple copies of the same tooltip:

```html live
<>
<u data-tip data-for="gobbledygook">
  blah blah blah
</u>

<br></br>

<u data-tip data-for="gobbledygook">complete and utter nonsense</u>

<br></br>

<u data-tip data-for="gobbledygook">entropy, yo</u>
</>
```

A couple notes on this:
- It seems that you have to set the same defaults on each of these tooltips, too, even if you already have on a `ReactTooltip` component *without* an `id`. This is obnoxious and seems wrong, but I'm not sure how to change it.
- You can use HTML in these tooltips. You're supposed to be able to in other ones by setting `html={true}`, but there must be a bug because this isn't working. Note that in the other ones you only need the `<br` tag for a newline, but with these you oddly enough need the closing tag too, as in `<br></br>`.

See the full list of options on the `react-tooltip` [npm page](https://www.npmjs.com/package/react-tooltip).
