## Additional Test Users

In addition to _Richard Hendricks_, we provide the following test users to help you implement specific flows and test error scenarios:

### `ALWAYS REQUEST DOB`

<TestUserDisplay
  data={{
    Phone: '+10019999999',
    'Birth Date': '1989-08-01',
    'Verification Code': '111111',
    'Error Code': '[OCE011](./errors#oce011)',
  }}
  description='Triggers Additional Information error to test Birth Date handling.'
></TestUserDisplay>

Use this test user to implement the Date of Birth (DoB) flow. Requests with this phone number trigger the [OCE011](./errors#oce011) error, requiring additional information (Birth Date). This allows testing and implementing error handling.

### `NO CREDENTIALS FOUND`

<TestUserDisplay
  data={{
    Phone: '+10019999998',
    'Error Code': '[OCE013](./errors#oce013)',
  }}
  description='Simulate no credentials found error.'
></TestUserDisplay>

This test user simulates the scenario where no credentials can be found for the provided phone number. Use this to implement proper error handling when the [OCE013](./errors#oce013) error is returned, ensuring your application gracefully handles cases where user data is unavailable.
