## `1ClickEntity`

```typescript title="1ClickEntity Type"
{
    "identifiers": {
        [identifierType: string]: string
    },
    "credentials": {
        [shortenedCredentialType: string]: string
    },
    "metadata": {
        "identifiers": {
            [identifierType: string]: {
                [metadataType: string]: string
            }
        },
        "credentials": {
            [shortenedCredentialType: string]: {
                [metadataType: string]: string
            }
        }
    }
}
```

### Metadata

> A piece of **metadata** is additional data that's not a user's identity data (their identifiers or credentials) — though it's often _about_ that identity data.

<table>
    <tr>
        <th>`metadataType`</th>
        <th>Type</th>
        <th>Format</th>
        <th>Description</th>
        <th>Example</th>
    </tr>
    <tr>
        <td>`id`</td>
        <td>`string`</td>
        <td>[Version 4 UUID](https://en.wikipedia.org/wiki/Universally_unique_identifier#Version_4_(random))</td>
        <td>Unique identifier for a credential (applies only to credentials, not identifiers)</td>
        <td>`"174b9aaf-3219-4499-b54c-0eb6acc5dbf2"`</td>
    </tr>
    <tr>
        <td>`verificationMethod`</td>
        <td>
            <details>
                <summary>`enum`</summary>

                Possible values:
                    - `"phone_carrier"`
                    - `"credit_bureau"`
                    - `"otp"`
                    - `"self_attested"`
                    - `null`
            </details>
        </td>
        <td>`snake_case`</td>
        <td>How an identifier or credential was verified</td>
        <td>`"phone_carrier"`</td>
    </tr>
    <tr>
        <td>`status`</td>
        <td>
            <details>
                <summary>`enum`</summary>

                Possible values:
                    - `"valid"`
                    - `"revoked"`
                    - `"otp"`
                    - `"self_attested"`
                    - `null`
            </details>
        </td>
        <td>`snake_case`</td>
        <td>The status of a credential</td>
        <td>`"valid"`</td>
    </tr>
    <tr>
        <td>`expirationDate`</td>
        <td>`string`</td>
        <td>[Unix time](https://en.wikipedia.org/wiki/Unix_time) (milliseconds) or `null`</td>
        <td>The date a credential expires</td>
        <td>`"1671847264479"`</td>
    </tr>
</table>

:::tip
**The most commonly used type of metadata is `verificationMethod`, which describes how a credential was verified:**
- `"phone_carrier"`: verified via phone carrier records
- `"credit_bureau"`: verified via credit bureau records
- `"otp"`: verified via an OTP (one time passcode)
- `"self_attested"`: attested by the user themselves (not verified)
- `null`

You can build custom logic to handle credentials based on their `verificationMethod`, for example requiring additional verification if an important credential is `"self_attested"`.
:::
