## `1ClickEntity`

```typescript title="1ClickEntity Type"
{
    "identifiers": {
        [identifierKey: string]: string
    },
    "credentials": {
        [credentialKey: string]: string
    },
    "metadata": {
        "identifiers": {
            [identifierKey: string]: {
                [metadataKey: string]: string
            }
        },
        "credentials": {
            [credentialKey: string]: {
                [metadataKey: string]: string
            }
        }
    }
}
```

<table>
    <tr>
        <th>Property</th>
        <th>Type</th>
        <th>Format</th>
        <th>Description</th>
        <th>Example</th>
    </tr>
    <tr>
        <td>`identifiers`</td>
        <td>`Object`</td>
        <td>See above</td>
        <td>User [identifiers](./data/#identifiers), keyed on `identifierKey`</td>
        <td>See [Example `1ClickEntity`](#example-1clickentity)</td>
    </tr>
    <tr>
        <td>`identifierKey`</td>
        <td>
            <details>
                <summary>`enum`</summary>

                Possible values (see [identifiers](./data/#identifiers) for details):
                    - `phone`
                    - `email`
            </details>
        </td>
        <td>`camelCase`</td>
        <td>The type of identifier</td>
        <td>`"phone"`</td>
    </tr>
    <tr>
        <td>`credentials`</td>
        <td>`Object`</td>
        <td>See above</td>
        <td>User [credentials](./data/#credentials), keyed on `credentialKey`</td>
        <td>See [Example `1ClickEntity`](#example-1clickentity)</td>
    </tr>
    <tr>
        <td>`credentialKey`</td>
        <td>
            <details>
                <summary>`enum`</summary>

                Possible values (see [credentials](./data/#credential) for details):
                    - `phone`
                    - `email`
                    - `fullName.firstName`
                    - `fullName.middleName`
                    - `fullName.lastName`
                    - `address.line1`
                    - `address.line2`
                    - `address.city`
                    - `address.state`
                    - `address.zipCode`
                    - `address.country`
                    - `birthDate`
                    - `ssn`
            </details>
        </td>
        <td>`camelCase`</td>
        <td>
            The type of user data a credential has
        </td>
        <td>`"firstName"`</td>
    </tr>
    <tr>
        <td>`metadata`</td>
        <td>`Object`</td>
        <td>See above</td>
        <td>[Metadata](./data/#metadata), keyed on `metadataKey`</td>
        <td>See [Example `1ClickEntity`](#example-1clickentity)</td>
    </tr>
    <tr>
        <td>`metadataKey`</td>
        <td>
            <details>
                <summary>`enum`</summary>

                Possible values:
                    - `id`
                    - `verificationMethod`
                    - `status`
                    - `issuanceDate`
                    - `expirationDate`
            </details>
        </td>
        <td>`camelCase`</td>
        <td>Any additional data that's not a user's identity data (identifiers or credentials)</td>
        <td>`"verificationMethod"`</td>
    </tr>
</table>
